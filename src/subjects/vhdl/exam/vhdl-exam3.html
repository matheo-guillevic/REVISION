Partie I : Partie cours [34/90]

Question 1.1

Énoncé : Codez en virgule fixe avec le minimum de bits 511,75 [6] .

Correction et Explications : Pour coder un nombre en virgule fixe, on convertit séparément la partie entière et la partie décimale en binaire.

  • Partie entière (511) : On sait que \(2^9 = 512\), donc \(511 = 2^9 - 1\). En binaire, 511 s'écrit avec 9 bits à '1' : \(111111111_2\).
  • Partie décimale (0,75) : \(0,75 = 0,5 + 0,25 = 2^{-1} + 2^{-2}\). En binaire, cela s'écrit \(0,11_2\).

Réponse :

  • En non signé, le minimum est de 11 bits : \(111111111,11_2\).
  • En signé (complément à 2), il faut ajouter un bit de signe (0 pour positif), soit un minimum de 12 bits : \(0111111111,11_2\).

Question 1.2

Énoncé : Simplifiez en utilisant un tableau de Karnaugh la fonction \(F\) [12] .

Note : Le texte d'origine a perdu les barres de négation (OCR : F=ABC+ABC+ABC+ABC+ABC ). Nous traiterons l'exemple suivant pour illustrer la méthode : \(F = \overline{A}\overline{B}C + \overline{A}BC + A\overline{B}C + AB\overline{C} + ABC\)

Correction et Explications : On place des '1' dans le tableau de Karnaugh pour chaque minterme :

\(A \backslash BC\) 00 01 11 10
0 0 1 (\(\overline{A}\overline{B}C\)) 1 (\(\overline{A}BC\)) 0
1 0 1 (\(A\overline{B}C\)) 1 (\(ABC\)) 1 (\(AB\overline{C}\))

Regroupements possibles (les plus grands possibles en puissances de 2) :

  • Un groupe de 4 (colonne 01 et 11) : donne \(C\).
  • Un groupe de 2 (ligne du bas, 11 et 10) : donne \(AB\).

Résultat simplifié : \(F = C + AB\).

Question 1.3

Énoncé : En VHDL, quel type normalisé doit être utilisé pour réaliser des opérations arithmétiques sur des vecteurs binaires non signés [2] ? Quels bibliothèques et paquetages déclarer [2] ? Montrez comment \(-5 - 7\) est obtenu en complément à 2 .

Correction et Explications :

  • Type normalisé : Il faut utiliser le type unsigned .
  • Bibliothèque et paquetage : library IEEE; use IEEE.numeric_std.all; (ne jamais utiliser l'obsolète std_logic_unsigned).
  • Opération \(-5 - 7\) en complément à 2 (sur 5 bits) :

Pour coder \(-12\) (le résultat), il faut au minimum 5 bits (plage de \(-16\) à \(+15\)).

  1. \(+5\) en binaire = \(00101_2\). Inversion = \(11010_2\). On ajoute 1 \(\rightarrow\) \(-5 = 11011_2\).
  2. \(+7\) en binaire = \(00111_2\). Inversion = \(11000_2\). On ajoute 1 \(\rightarrow\) \(-7 = 11001_2\).
  3. Addition : \(11011_2 + 11001_2 = (1) 10100_2\). On ignore la retenue sortante.
  4. Résultat : \(10100_2\). Pour vérifier sa valeur : on inverse (\(01011_2\)) et on ajoute 1 (\(01100_2 = 12_{10}\)). Le résultat est bien \(-12\).

Question 1.4

Énoncé : En VHDL est-ce que l'ordre des processus a une importance ? Si non, dans quel ordre sont-ils exécutés ?

Correction et Explications : Non, l'ordre des processus dans l'architecture n'a aucune importance . En VHDL, les processus sont des instructions concurrentes. Ils s'exécutent de manière parallèle . Leur exécution est déclenchée (réveillée) uniquement par les événements qui surviennent sur les signaux présents dans leur liste de sensibilité.

Partie II

Partie II : Conception d'un registre à décalage SIPO [56/90]

Question 2.1

Énoncé : Dessinez le schéma d'un registre SIPO de \(n\) bits en utilisant \(n\) bascules D . Entrées : din, clk, enable, reset . Sortie : dout (n bits) .

(Voir le code SVG fourni séparément pour générer ou visualiser ce schéma). Explication : Les bascules sont en cascade. L'entrée \(D\) de la première bascule est liée à din. L'entrée \(D\) des bascules suivantes est liée à la sortie \(Q\) de la bascule précédente. clk, reset et enable sont communs à toutes les bascules.

Question 2.2

Énoncé : Ecrire en VHDL RTL l'entité générique (shift) et l'architecture (rtl) .

Correction :

library IEEE;
use IEEE.std_logic_1164.all;

entity shift is
    generic (n : integer := 8);
    port (
        din    : in  std_logic;
        clk    : in  std_logic;
        reset  : in  std_logic;
        enable : in  std_logic;
        dout   : out std_logic_vector(n-1 downto 0)
    );
end entity shift;

architecture rtl of shift is
    signal shift_reg : std_logic_vector(n-1 downto 0);
begin
    process(clk)
    begin
        if rising_edge(clk) then
            if reset = '1' then
                shift_reg <= (others => '0');
            elsif enable = '1' then
                -- Decalage vers la gauche : le MSB recoit din
                shift_reg <= shift_reg(n-2 downto 0) & din;
            end if;
        end if;
    end process;
    dout <= shift_reg;
end architecture rtl;

Question 2.3

Énoncé : Ecrire en VHDL structurel l'architecture (struct) de la même entité . Composant disponible : bascule .

Correction :

architecture struct of shift is
    component bascule
        port(d, clk, reset, enable: in std_logic;
             q: out std_logic);
    end component;
    
    -- Signal intermediaire pour relier les bascules en cascade
    signal q_int : std_logic_vector(n downto 0);
begin
    q_int(0) <= din; -- L'entree du registre attaque la 1ere bascule

    gen_shift: for i in 0 to n-1 generate
        bascule_inst : bascule port map (
            d      => q_int(i),
            clk    => clk,
            reset  => reset,
            enable => enable,
            q      => q_int(i+1)
        );
    end generate gen_shift;

    -- Affectation des sorties paralleles
    dout <= q_int(n downto 1);
end architecture struct;

Énoncé : Ecrire un testbench simulant l'architecture structurelle pour \(n=8\) .

Correction :

library IEEE;
use IEEE.std_logic_1164.all;

entity tb_shift is
end entity tb_shift;

architecture sim of tb_shift is
    signal din, clk, reset, enable : std_logic := '0';
    signal dout : std_logic_vector(7 downto 0);
    constant period : time := 10 ns;
begin
    UUT: entity work.shift(struct)
        generic map (n => 8)
        port map (din => din, clk => clk, reset => reset, enable => enable, dout => dout);

    clk_proc: process begin
        clk <= '0'; wait for period/2;
        clk <= '1'; wait for period/2;
    end process;

    stim_proc: process begin
        reset <= '1'; wait for 20 ns;
        reset <= '0'; enable <= '1'; din <= '1'; wait for period;
        din <= '0'; wait for 2 * period;
        enable <= '0'; wait for 2 * period; -- Retenue d'etat
        enable <= '1'; din <= '1'; wait for 2 * period;
        reset <= '1'; wait for period;
        wait;
    end process;
end architecture sim;
