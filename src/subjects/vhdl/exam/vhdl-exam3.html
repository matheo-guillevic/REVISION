Problème I - Partie cours

1.1 Codez en virgule fixe avec le minimum de bits 511,75 [6].

Correction et raisonnement

Étape 1 : Conversion de la partie entière.
On convertit la valeur entière 511 en binaire. Sachant que \(2^9 = 512\), on a \(511 = 2^9 - 1\). En binaire, cela correspond à 9 bits tous à '1' : \[511_{10} = 111111111_2\]

Étape 2 : Conversion de la partie décimale.
On convertit 0,75 en sommes de puissances négatives de 2 : \(0,75 = 0,5 + 0,25 = 2^{-1} + 2^{-2}\). \[0,75_{10} = 0,11_2\]

Étape 3 : Format non signé.
En combinant les deux parties, on obtient le code en virgule fixe non signé. Il nécessite au minimum 11 bits : \[\mathbf{111111111,11_2}\]

Étape 4 : Format signé (complément à 2).
Pour une représentation signée, un bit de signe doit être ajouté en position de poids fort. Le nombre étant strictement positif, le bit de signe est '0'. Cela requiert au minimum 12 bits : \[\mathbf{0111111111,11_2}\]

1.2 Simplifiez en utilisant un tableau de Karnaugh la fonction \(F = \overline{A}\overline{B}C + \overline{A}BC + A\overline{B}C + AB\overline{C} + ABC\) [12].

Correction et raisonnement

Étape 1 : Remplissage du tableau de Karnaugh.
Pour la fonction donnée, on place les '1' dans les cases correspondant aux mintermes :

\(A \backslash BC\) 00 01 11 10
0 0 1 (\(\overline{A}\overline{B}C\)) 1 (\(\overline{A}BC\)) 0
1 0 1 (\(A\overline{B}C\)) 1 (\(ABC\)) 1 (\(AB\overline{C}\))

Étape 2 : Regroupement des mintermes.
On cherche à faire les plus grands groupements adjacents possibles (tailles en puissances de 2) :

  • Un groupe de 4 cases (colonnes 01 et 11) qui isole la variable \(C\).
  • Un groupe de 2 cases (ligne \(A=1\), colonnes 11 et 10) qui isole \(AB\).

Étape 3 : Expression simplifiée.
La fonction finale est la somme logique de ces groupements : \[\mathbf{F = C + AB}\]

1.3 En VHDL, quel type normalisé doit être utilisé pour réaliser des opérations arithmétiques sur des vecteurs binaires non signés [2] ? Quels bibliothèques et paquetages déclarer [2] ? Montrez comment \(-5 - 7\) est obtenu en complément à 2.

Correction et raisonnement

Étape 1 : Type normalisé et bibliothèques.
Pour réaliser des opérations arithmétiques sur des vecteurs non signés, le type normalisé à utiliser est unsigned. Il faut déclarer la bibliothèque IEEE et son paquetage mathématique officiel : \[\mathbf{\texttt{library IEEE; use IEEE.numeric\_std.all;}}\]

Étape 2 : Préparation des opérandes sur 5 bits.
Pour que le résultat (\(-12\)) puisse être codé sans débordement, il faut au minimum 5 bits (plage de \(-16\) à \(+15\)).

  • \(+5 = 00101_2 \implies \text{Inversion} = 11010_2 \implies \text{Ajout de 1} \implies \mathbf{-5 = 11011_2}\)
  • \(+7 = 00111_2 \implies \text{Inversion} = 11000_2 \implies \text{Ajout de 1} \implies \mathbf{-7 = 11001_2}\)

Étape 3 : Addition en complément à 2.
On effectue l'addition binaire standard de ces deux nombres négatifs et on ignore la retenue sortante du bit de poids fort : \[11011_2 + 11001_2 = (1) 10100_2\]

Étape 4 : Résultat final.
Le résultat est : \[\mathbf{10100_2}\] (Vérification : l'inverse est \(01011_2\), plus 1 donne \(01100_2 = 12_{10}\). Le résultat est donc bien \(-12\)).

1.4 En VHDL est-ce que l'ordre des processus a une importance ? Si non, dans quel ordre sont-ils exécutés ?

Correction et raisonnement

Étape 1 : Importance de l'ordre structurel.
Non, l'ordre physique d'écriture des processus dans l'architecture n'a absolument aucune importance. Le VHDL modélise des composants matériels.

Étape 2 : Mécanisme d'exécution réel.
Les processus s'exécutent de manière concurrente (parallèle). Leur exécution est purement événementielle : un processus est évalué uniquement lorsqu'un changement d'état (événement) se produit sur l'un des signaux inscrits dans sa liste de sensibilité.

Probleme 2

Problème II - Conception d'un registre à décalage SIPO

2.1 Dessinez le schéma d'un registre SIPO de \(n\) bits en utilisant \(n\) bascules D. Entrées : din, clk, enable, reset. Sortie : dout (n bits).

Correction et raisonnement

Étape 1 : Structure générale.
Un registre à décalage SIPO (Serial In, Parallel Out) de \(n\) bits est constitué de \(n\) bascules D identiques, connectées en cascade pour faire transiter l'information bit par bit.

Étape 2 : Connexions.

  • L'entrée de données série din attaque l'entrée D de la première bascule.
  • La sortie Q d'une bascule de rang \(i\) attaque l'entrée D de la bascule suivante de rang \(i+1\).
  • Les signaux de contrôle (clk, reset, enable) sont connectés à toutes les bascules en parallèle.
  • Le vecteur de sortie dout regroupe en parallèle les \(n\) sorties Q des différentes bascules.

2.2 Ecrire en VHDL RTL l'entité générique (shift) et l'architecture (rtl).

Correction et raisonnement

Étape 1 : Définition de l'entité avec generic.
On déclare les signaux de port, ainsi que le paramètre n.

Étape 2 : Processus de décalage.
On crée un processus synchrone pour décrire le comportement RTL, en gérant le reset et le signal enable.

library IEEE;
use IEEE.std_logic_1164.all;

entity shift is
    generic (n : integer := 8);
    port (
        din    : in  std_logic;
        clk    : in  std_logic;
        reset  : in  std_logic;
        enable : in  std_logic;
        dout   : out std_logic_vector(n-1 downto 0)
    );
end entity shift;

architecture rtl of shift is
    signal shift_reg : std_logic_vector(n-1 downto 0);
begin
    process(clk)
    begin
        if rising_edge(clk) then
            if reset = '1' then
                shift_reg <= (others => '0');
            elsif enable = '1' then
                -- Decalage : le LSB glisse, din entre en position 0
                shift_reg <= shift_reg(n-2 downto 0) & din;
            end if;
        end if;
    end process;
    dout <= shift_reg;
end architecture rtl;

2.3 Ecrire en VHDL structurel l'architecture (struct) de la même entité. Composant disponible : bascule.

Correction et raisonnement

Étape 1 : Déclaration des composants et signaux internes.
On déclare le composant unitaire bascule et un vecteur de signaux internes q_int pour relier les bascules entre elles.

Étape 2 : Instanciation avec une boucle.
On utilise l'instruction for ... generate pour instancier les \(n\) bascules et on assigne les câblages matériels de cascade.

architecture struct of shift is
    component bascule
        port(d, clk, reset, enable: in std_logic;
             q: out std_logic);
    end component;
    
    signal q_int : std_logic_vector(n downto 0);
begin
    q_int(0) <= din;

    gen_shift: for i in 0 to n-1 generate
        bascule_inst : bascule port map (
            d      => q_int(i),
            clk    => clk,
            reset  => reset,
            enable => enable,
            q      => q_int(i+1)
        );
    end generate gen_shift;

    dout <= q_int(n downto 1);
end architecture struct;

2.4 Ecrire un testbench simulant l'architecture structurelle pour \(n=8\).

Correction et raisonnement

Étape 1 : Instanciation de l'UUT et de l'horloge.
On déclare l'entité de test, les signaux, on mappe les ports et on crée un processus autonome de génération de la porteuse d'horloge.

Étape 2 : Scénario d'excitation.
On crée un processus pour jouer une séquence (reset, entrée de 1, désactivation, etc.).

library IEEE;
use IEEE.std_logic_1164.all;

entity tb_shift is
end entity tb_shift;

architecture sim of tb_shift is
    signal din, clk, reset, enable : std_logic := '0';
    signal dout : std_logic_vector(7 downto 0);
    constant period : time := 10 ns;
begin
    UUT: entity work.shift(struct)
        generic map (n => 8)
        port map (din => din, clk => clk, reset => reset, enable => enable, dout => dout);

    clk_proc: process begin
        clk <= '0'; wait for period/2;
        clk <= '1'; wait for period/2;
    end process;

    stim_proc: process begin
        reset <= '1'; wait for 20 ns;
        reset <= '0'; enable <= '1'; din <= '1'; wait for period;
        din <= '0'; wait for 2 * period;
        enable <= '0'; wait for 2 * period; -- Retenue d'etat
        enable <= '1'; din <= '1'; wait for 2 * period;
        reset <= '1'; wait for period;
        wait;
    end process;
end architecture sim;

2.5 Dessinez les chronogrammes correspondant au testbench.

Correction et raisonnement

Étape 1 : Analyse des signaux d'entrée.
D'après le testbench, l'information din est insérée bit par bit au rythme de l'horloge, à condition que enable soit à '1'.

Étape 2 : Tracé des sorties.
