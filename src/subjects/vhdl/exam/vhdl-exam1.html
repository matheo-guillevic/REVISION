Partie I : Conception d'un registre [40/90]

Question 1.1

Énoncé : Dessinez en utilisant uniquement des bascules DFF (possédant des entrées reset et enable) le circuit d'un registre \(n\) bits à chargement parallèle et sortie parallèle capturant de l'entrée Din sur la sortie Dout sur les fronts montants de l'horloge clk avec :

  • une entrée enable active à l'état haut autorisant le chargement de l'entrée Din .
  • une entrée de données Din sur \(n\) bits .
  • une sortie Dout sur \(n\) bits .
  • une remise à zéro Reset synchrone active à l'état haut .

Correction et Explication : Un registre sur \(n\) bits est simplement une mise en parallèle de \(n\) bascules D. Pour concevoir ce circuit :

  • Tous les signaux de contrôle (clk, reset, enable) sont connectés en commun à toutes les bascules.
  • Le bit \(i\) de l'entrée de données (Din[i]) est connecté à l'entrée D de la bascule \(i\).
  • Le bit \(i\) de la sortie de données (Dout[i]) est connecté à la sortie Q de la bascule \(i\).

Note : Lors de la capture d'un front montant d'horloge, si reset est à '1', la sortie passe à 0. Sinon, si enable est à '1', l'entrée D est recopiée sur la sortie Q.

Question 1.2

Énoncé : Ecrire en VHDL RTL l'entité générique (registerN) et l'architecture (rtl) du composant précédent [10] .

Correction et Explication : La description "RTL" (Register Transfer Level) décrit le comportement du registre en utilisant un processus synchrone. Le reset étant synchrone, il doit être testé à l'intérieur de la condition du front d'horloge .

library IEEE;
use IEEE.std_logic_1164.all;

entity registerN is
    generic (
        n : integer := 8 -- Valeur par defaut arbitraire
    );
    port (
        clk    : in  std_logic;
        reset  : in  std_logic;
        enable : in  std_logic;
        Din    : in  std_logic_vector(n-1 downto 0);
        Dout   : out std_logic_vector(n-1 downto 0)
    );
end entity registerN;

architecture rtl of registerN is
begin
    process(clk)
    begin
        if rising_edge(clk) then
            if reset = '1' then
                Dout <= (others => '0'); -- Remise a zero synchrone
            elsif enable = '1' then
                Dout <= Din;             -- Chargement parallele
            end if;
        end if;
    end process;
end architecture rtl;

Question 1.3

Énoncé : Combien de bascules seront générées par la synthèse de cette description rtl [5] .

Correction et Explication : La synthèse logique générera exactement \(n\) bascules D. Explication : L'entrée et la sortie étant définies par des vecteurs de taille \(n\) (n-1 downto 0), le processus assigne une valeur mémorisée à chaque bit du vecteur Dout à chaque front d'horloge. Chaque bit mémorisé correspond physiquement à une bascule.

Question 1.4

Énoncé : Ecrire en VHDL structurel l'architecture (struct) de la même entité (registerN) précédente [15] . On dispose du composant suivant :

entity bascule is
port(d, clk, reset, enable: in std_logic;
     q: out std_logic);
end entity bascule;

Correction et Explication : Une description structurelle consiste à instancier et relier des sous-composants existants (ici, l'entité bascule). Puisque le registre a une taille générique \(n\), nous devons utiliser une boucle generate pour instancier les \(n\) bascules de façon élégante.

architecture struct of registerN is
    -- Declaration du composant a instancier
    component bascule
        port(d, clk, reset, enable: in std_logic;
             q: out std_logic);
    end component;
begin
    -- Generation de n bascules
    gen_bascules: for i in 0 to n-1 generate
        bascule_inst : bascule
            port map (
                d      => Din(i),
                clk    => clk,
                reset  => reset,
                enable => enable,
                q      => Dout(i)
            );
    end generate gen_bascules;
end architecture struct;
Partie II

Partie II : Conception d'un « compteur générique » [50/90]

Question 2.1

Énoncé : Ecrire en VHDL RTL l'entité (compteur_moduloN) avec un paramètre générique N et l'architecture (rtl) d'un compteur comptant sur 4 bits et modulo N [15] .

  • Entrées : horloge (clk) sur front montant, reset (rst) asynchrone actif haut, enable (en) actif haut .
  • Sorties : valeur (count, sur 4 bits), signal (fin) qui passe à '1' quand la valeur max (N-1) est atteinte .

Correction et Explication : Le compteur est "modulo N", ce qui signifie qu'il compte de \(0\) à \(N-1\), puis repasse à \(0\). Le reset asynchrone implique que le signal rst doit figurer dans la liste de sensibilité du processus et être testé avant le front d'horloge.

library IEEE;
use IEEE.std_logic_1164.all;
use IEEE.numeric_std.all; -- Utilisation recommandee pour les compteurs

entity compteur_moduloN is
    generic (
        N : integer := 10 -- Modulo par defaut
    );
    port (
        clk   : in  std_logic;
        rst   : in  std_logic;
        en    : in  std_logic;
        count : out std_logic_vector(3 downto 0);
        fin   : out std_logic
    );
end entity compteur_moduloN;

architecture rtl of compteur_moduloN is
    -- Signal interne pour faciliter les operations arithmetiques
    signal count_int : unsigned(3 downto 0);
begin
    process(clk, rst)
    begin
        -- Reset asynchrone
        if rst = '1' then
            count_int <= (others => '0');
        -- Action synchrone sur front montant
        elsif rising_edge(clk) then
            if en = '1' then
                if count_int = N - 1 then
                    count_int <= (others => '0'); -- Bouclage modulo N
                else
                    count_int <= count_int + 1;   -- Incrementation
                end if;
            end if;
        end if;
    end process;

    -- Assignations concurrentes pour les sorties
    count <= std_logic_vector(count_int);
    
    -- Le signal fin vaut '1' uniquement quand le compteur atteint N-1
    fin <= '1' when (count_int = N - 1) else '0';
    
end architecture rtl;

Question 2.2

Énoncé : Combien de bascules seront générées par la synthèse logique de votre description précédente [5] ?

Correction et Explication : 4 bascules seront générées. Explication : L'énoncé précise explicitement que la sortie du compteur (count) est sur 4 bits . Même si \(N\) est générique, la taille du registre interne mémorisant l'état est fixée à 4 bits (de 0 à 15 maximum). Le signal fin est un signal purement combinatoire généré à partir de ces 4 bascules (il n'est pas mémorisé).

Question 2.3

Énoncé : Ecrire en VHDL un testbench permettant de simuler l'architecture précédente pour un modulo \(N=8\) [15] .

Correction et Explication : Un banc de test (testbench) n'a ni entrée ni sortie. Il instancie le composant à tester (UUT - Unit Under Test), génère l'horloge et applique des stimuli (scénario de test) sur les entrées pour observer le comportement.

library IEEE;
use IEEE.std_logic_1164.all;

entity tb_compteur is
-- Un testbench n'a pas de ports
end entity tb_compteur;

architecture behavior of tb_compteur is
    -- Signaux pour se connecter a l'UUT
    signal clk   : std_logic := '0';
    signal rst   : std_logic := '0';
    signal en    : std_logic := '0';
    signal count : std_logic_vector(3 downto 0);
    signal fin   : std_logic;

    constant clk_period : time := 10 ns;

begin
    -- Instanciation de l'UUT avec N = 8
    uut: entity work.compteur_moduloN
        generic map (
            N => 8
        )
        port map (
            clk   => clk,
            rst   => rst,
            en    => en,
            count => count,
            fin   => fin
        );

    -- Generation de l'horloge
    clk_process : process
    begin
        clk <= '0';
        wait for clk_period/2;
        clk <= '1';
        wait for clk_period/2;
    end process;

    -- Processus des stimuli
    stim_proc: process
    begin
        -- Initialisation : Reset actif
        rst <= '1';
        en  <= '0';
        wait for 20 ns;
        
        -- Relachement du reset, activation du compteur
        rst <= '0';
        en  <= '1';
        
        -- On laisse tourner le compteur pour voir au moins un cycle complet (N=8)
        -- 8 coups d'horloge font 80 ns. On attend 100 ns.
        wait for 100 ns;
        
        -- Fin de la simulation
        en <= '0';
        wait;
    end process;
end architecture behavior;

Question 2.4

Énoncé : Dessinez les chronogrammes des signaux de l'entité correspondant très exactement à la simulation que donnerait votre testbench précédent [15] . Indiquez clairement l'échelle des temps sur l'axe temporel .

Correction et Explication : Voici la description textuelle du chronogramme généré par le banc de test précédent.

  • De 0 ns à 20 ns :
  • rst = '1', en = '0'
  • count = "0000" (0 en décimal, maintenu par le reset asynchrone)
  • fin = '0'
  • À 20 ns : rst descend à '0', en passe à '1'.
  • De 20 ns à 100 ns (Activité de comptage) :
  • À 25 ns (front montant) : count \(\rightarrow\) 1
  • À 35 ns (front montant) : count \(\rightarrow\) 2
  • À 45 ns (front montant) : count \(\rightarrow\) 3
  • À 55 ns (front montant) : count \(\rightarrow\) 4
  • À 65 ns (front montant) : count \(\rightarrow\) 5
  • À 75 ns (front montant) : count \(\rightarrow\) 6
  • À 85 ns (front montant) : count \(\rightarrow\) 7. Le signal fin passe immédiatement à '1' (logique combinatoire).
  • À 95 ns (front montant) : count \(\rightarrow\) 0 (bouclage car \(N=8\)). Le signal fin retombe immédiatement à '0'.
  • Au-delà de 100 ns :
  • À 105 ns (front montant) : count \(\rightarrow\) 1.
  • À 120 ns, en repasse à '0' (selon le testbench), le compteur s'arrête sur sa valeur courante.
