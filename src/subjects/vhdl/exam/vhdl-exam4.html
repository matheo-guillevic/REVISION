Partie I

Questions de cours

1.1 Type VHDL pour les vecteurs signes et calcul de \(-6 \times 7\)

Pour faire de l'arithmetique signee normalisee sur des vecteurs binaires en complement a 2, il faut utiliser le type signed du paquetage ieee.numeric_std.

library ieee;
use ieee.std_logic_1164.all;
use ieee.numeric_std.all;

Sur 4 bits, \(7=0111_2\). Pour \(-6\), on part de \(6=0110_2\), on inverse puis on ajoute 1 : \(1001+1=1010_2\).

Le resultat doit etre conserve sur 8 bits. En decimal, \(-6 \times 7=-42\). Le codage de \(42\) sur 8 bits vaut \(00101010_2\). Son complement a 2 vaut :

\(00101010 \rightarrow 11010101+1=11010110\)

Donc le resultat attendu est \(11010110_2=-42\).

1.3 Effet de wait; dans un process

  • Avec wait; inconditionnel : le processus s'execute une fois puis reste bloque pour toujours. C'est classique dans un testbench.
  • Sans wait et sans liste de sensibilite : le process boucle immediatement en cycles delta, sans faire avancer le temps de simulation.

1.4 Ordre des processus

L'ordre d'ecriture des processus dans le fichier VHDL n'a pas d'importance. Les processus sont concurrents : ils representent des blocs materiels qui existent en meme temps. En simulation, un processus est reveille par les evenements des signaux presents dans sa liste de sensibilite, puis ses instructions internes s'executent sequentiellement.

1.5 Traductions Verilog vers VHDL

Affectation continue :

x <= a and b and c;

Multiplexeur combinatoire :

process(ad, e0, e1, e2, e3)
begin
  case ad is
    when "00" => s <= e0;
    when "01" => s <= e1;
    when "10" => s <= e2;
    when others => s <= e3;
  end case;
end process;

Verrou combinatoire volontaire, car il n'y a pas de else :

process(c, d)
begin
  if c = '1' then
    q <= d;
  end if;
end process;

Registre a decalage SISO/SIPO :

library ieee;
use ieee.std_logic_1164.all;

entity SISO_SIPO is
  port (
    clk, d : in std_logic;
    s      : out std_logic;
    q      : out std_logic_vector(4 downto 1)
  );
end entity;

architecture rtl of SISO_SIPO is
  signal q_t : std_logic_vector(4 downto 1);
begin
  process(clk)
  begin
    if rising_edge(clk) then
      q_t <= q_t(3 downto 1) & d;
    end if;
  end process;

  q <= q_t;
  s <= q_t(4);
end architecture;
Partie II

Analyse d'une description VHDL : PWM

Architecture analysee

L'architecture correspond a un compteur 4 bits et a une sortie tamponnee. Le reset du sujet est asynchrone, car il est teste avant le front d'horloge.

architecture rtl of hidden is
  signal r   : unsigned(3 downto 0);
  signal buf : std_logic;
begin
  process(clk, reset)
  begin
    if reset = '1' then
      r   <= (others => '0');
      buf <= '0';
    elsif rising_edge(clk) then
      r <= r + 1;
      if (r < unsigned(w)) or (w = "0000") then
        buf <= '1';
      else
        buf <= '0';
      end if;
    end if;
  end process;

  pulse <= buf;
end architecture;

2.1 Reset synchrone ou asynchrone ?

Il est asynchrone. La condition reset = '1' est evaluee directement dans le process, avant le test rising_edge(clk). Le circuit peut donc etre remis a zero sans attendre un front montant.

2.2 Version avec reset synchrone

process(clk)
begin
  if rising_edge(clk) then
    if reset = '1' then
      r   <= (others => '0');
      buf <= '0';
    else
      r <= r + 1;
      if (r < unsigned(w)) or (w = "0000") then
        buf <= '1';
      else
        buf <= '0';
      end if;
    end if;
  end if;
end process;

2.6 Nombre de bascules synthetisees

Il y a 5 bascules : 4 pour le registre r de type unsigned(3 downto 0), et 1 pour buf. Le signal pulse est seulement une affectation concurrente de buf, donc un fil.

2.7 Fonction realisee

Cette architecture realise un generateur PWM, ou modulation de largeur d'impulsion. Le compteur \(r\) donne une periode fixe de 16 cycles ; \(w\) fixe le rapport cyclique. Plus \(w\) est grand, plus pulse reste haut longtemps. Cas particulier : si \(w=0\), la condition w = "0000" force pulse a 1.

Partie III

Machine a etats de Moore

3.1 Diagramme de transitions

La machine doit detecter le passage \(0 \rightarrow 1\) de \(X\), mettre \(Y=1\) pendant deux cycles, puis attendre que \(X\) retombe a 0 avant de pouvoir redemarrer.

3.2 Chronogramme

La sortie reagit au front d'horloge qui suit la montee de \(X\), puis reste a 1 pendant les etats \(S1\) et \(S2\).

3.3 Code VHDL complet

library ieee;
use ieee.std_logic_1164.all;

entity edge_fsm is
  port (
    clk : in std_logic;
    rst : in std_logic;
    X   : in std_logic;
    Y   : out std_logic
  );
end entity;

architecture rtl of edge_fsm is
  type state_t is (S0, S1, S2, S3);
  signal state, next_state : state_t;
begin
  process(clk)
  begin
    if rising_edge(clk) then
      if rst = '1' then
        state <= S0;
      else
        state <= next_state;
      end if;
    end if;
  end process;

  process(state, X)
  begin
    next_state <= state;
    Y <= '0';

    case state is
      when S0 =>
        if X = '1' then
          next_state <= S1;
        end if;

      when S1 =>
        Y <= '1';
        next_state <= S2;

      when S2 =>
        Y <= '1';
        if X = '1' then
          next_state <= S3;
        else
          next_state <= S0;
        end if;

      when S3 =>
        if X = '0' then
          next_state <= S0;
        end if;
    end case;
  end process;
end architecture;

3.4 Nombre minimal de bascules

La FSM possede 4 etats. Le nombre minimal de bits pour coder 4 etats est :

\(n=\lceil \log_2(4)\rceil=2\)

Il faut donc au minimum 2 bascules pour le registre d'etat, en supposant un codage binaire dense.
