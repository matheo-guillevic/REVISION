Exercice 1

Exercice 1 (2 points)

Question a

Indiquer la ligne mise en cause et en expliquer la raison.

Réponse et Raisonnement

L'erreur de compilation se trouve à la ligne : Math m = new Math();. La raison est que le constructeur de la classe Math est explicitement déclaré comme privé (\texttt{private Math() \{\}}). Il est donc impossible d'instancier un objet de cette classe depuis l'extérieur. De plus, PI est une variable statique.

Question b

Proposer un correctif.

Réponse

Puisque la constante PI est statique, on doit l'appeler directement depuis la classe.

double r = 10;
double perimetre = 2 * Math.PI * r;

Exercice 4

Exercice 4 (2 points)

Question

Expliquer la différence en Java entre int et Integer. Quel nom est donné aux classes Integer, Double, Boolean ? Quel est l'intérêt ?

Réponse et Raisonnement
  • Différence : int est un type primitif (valeur stockée en mémoire), tandis que Integer est un objet (référence vers un espace mémoire encapsulant un entier).
  • Nom : Ce sont des classes enveloppes (ou Wrapper classes en anglais).
  • Intérêt : Elles permettent de manipuler des types primitifs comme des objets. C'est indispensable pour utiliser les collections génériques en Java (comme ArrayList<Integer>) qui n'acceptent que des objets. De plus, ces classes fournissent des méthodes utilitaires pratiques (ex: Integer.parseInt()).
Exercice 5

Exercice 5 (3 points)

Question a

Différence entre redéfinition et surcharge.

Réponse
  • Redéfinition (Overriding) : Consiste à réécrire une méthode héritée d'une classe mère en gardant exactement la même signature (nom, type de retour et paramètres).
  • Surcharge (Overloading) : Consiste à créer plusieurs méthodes portant le même nom mais avec des paramètres différents (nombre ou type) au sein de la même classe (ou héritée).

Question b

Dans MaterielSki, indiquer la méthode redéfinie et surchargée.

Réponse
  • public double getTarifJournee() est la méthode redéfinie (elle écrase celle de Materiel).
  • public double getTarifJournee(double coeff) est la méthode surchargée.

Exercice 6 (3 points)

Question 1

Indiquer les erreurs et expliquer la raison.

Réponse et Raisonnement
  • La méthode getMarque() est déclarée abstract mais possède un corps (des accolades avec un return). Une méthode abstraite ne doit avoir qu'une signature terminée par un point-virgule.
  • La classe Materiel contient une méthode abstraite, elle doit donc obligatoirement être déclarée abstraite (public abstract class Materiel).

Question 2

Apporter un correctif.

Réponse

L'attribut marque étant présent dans la classe, le plus logique n'est pas de faire une méthode abstraite, mais une méthode concrète classique. On retire simplement le mot-clé abstract.

public String getMarque() {
    return marque;
}
Exercice 7

Exercice 7 (2 points)

Question

Quel est le résultat de ce programme ?

Réponse et Raisonnement

Grâce au polymorphisme (liaison dynamique), la méthode appelée est toujours celle du type réel de l'objet instancié en mémoire. Si la méthode n'est pas redéfinie, c'est celle de la classe mère qui est utilisée.

  • a = new A(); a.affiche(); \(\rightarrow\) Je suis un A
  • b = new B(); b.affiche(); \(\rightarrow\) Je suis un A (B hérite de A, pas de redéfinition)
  • a = b; a.affiche(); \(\rightarrow\) Je suis un A
  • c = new C(); c.affiche(); \(\rightarrow\) Je suis un C (C redéfinit la méthode)
  • a = c; a.affiche(); \(\rightarrow\) Je suis un C
  • d = new D(); d.affiche(); \(\rightarrow\) Je suis un D (D redéfinit)
  • a = d; a.affiche(); c = d; c.affiche(); \(\rightarrow\) Je suis un D
    Je suis un D
  • e = new E(); e.affiche(); \(\rightarrow\) Je suis un A (E hérite de B qui hérite de A)
  • a = e; a.affiche(); b = e; b.affiche(); \(\rightarrow\) Je suis un A
    Je suis un A
  • f = new F(); f.affiche(); \(\rightarrow\) Je suis un C (F hérite de C qui a sa redéfinition)
  • a = f; a.affiche(); c = f; c.affiche(); \(\rightarrow\) Je suis un C
    Je suis un C

Exercice 9

Exercice 9 (3 points)

Question 1

Erreur à l'exécution et correctif.

Réponse et Raisonnement

Le programme lance une NullPointerException car la liste lesMateriels est déclarée mais n'est jamais instanciée. Elle vaut null au moment de l'appel lesMateriels.add(...).

ArrayList<Materiel> lesMateriels = new ArrayList<>();

Question 2

Afficher le catalogue en appliquant 20\% de réduction sur les skis.

Réponse et Raisonnement

On utilise instanceof pour vérifier si l'objet est un équipement de ski. Si c'est le cas, on le caste pour appeler la méthode surchargée avec le coefficient 0.8 (ce qui équivaut à -20\%).

for (Materiel m : lesMateriels) {
    if (m instanceof MaterielSki) {
        MaterielSki ms = (MaterielSki) m;
        // La methode surchagee applique un multiplicateur (ex: 0.8 pour -20%)
        double tarifPromo = ms.getTarifJournee(0.8);
        System.out.println(ms.toString() + " PROMO!!" + tarifPromo);
    } else {
        System.out.println(m.toString());
    }
}
