Probleme 1

Problème 1 - Gestion de location

Vous êtes loueur de voiture. Vous souhaitez informatiser la gestion de votre magasin. Chaque véhicule possède 3 caractéristiques : un numéro d'immatriculation (9 caractères, unique), un poids en kg, et un nombre total de places. Vous louez 3 types de véhicules : des voitures (on gère le nombre de places enfants), des camions (on gère la charge utile en kg) et des motos (on gère la cylindrée). Chaque véhicule appartient à une catégorie (qui détermine le prix à la journée).

Questions 1 et 2

Donnez le code des classes Categorie, Vehicule, Camion, Moto, Voiture. La classe Vehicule doit contenir une méthode getPrixLocation(int nbJour).

Correction et raisonnement

Étape 1 : Création de la classe Categorie.
Il s'agit d'une classe simple agissant comme un conteneur de données pour le nom et le prix.

public class Categorie {
    private String nom;
    private double prixJour;

    public Categorie(String nom, double prixJour) {
        this.nom = nom;
        this.prixJour = prixJour;
    }
    public double getPrixJour() { return prixJour; }
}
Correction et raisonnement

Étape 2 : Création de la super-classe Vehicule.
On la déclare abstraite car un "Véhicule" générique n'est pas censé être instancié directement.

public abstract class Vehicule {
    protected String immatriculation;
    protected double poids;
    protected int nbPlaces;
    protected Categorie categorie;

    public Vehicule(String imm, double p, int nb, Categorie c) {
        this.immatriculation = imm; 
        this.poids = p; 
        this.nbPlaces = nb; 
        this.categorie = c;
    }

    public String getImmatriculation() { return immatriculation; }
    
    public double getPrixLocation(int nbJour) { 
        return categorie.getPrixJour() * nbJour; 
    }
}
Correction et raisonnement

Étape 3 : Création des classes filles.
Elles héritent de Vehicule et utilisent super() pour initialiser les attributs communs.

public class Voiture extends Vehicule {
    private int placesEnfants;

    public Voiture(String imm, double p, int nb, Categorie c, int pe) {
        super(imm, p, nb, c); 
        this.placesEnfants = pe;
    }
}

Question 3

Créez une classe Parc possédant les méthodes addVehicule, removeVehicule, displayParc et getPrixTotalLocation. Les ajouts/suppressions doivent gérer les exceptions en cas d'erreurs (doublons ou introuvables).

Correction et raisonnement

Étape 1 : Implémentation du Parc et gestion de la collection.
On utilise une ArrayList pour stocker les objets.

import java.util.ArrayList;

public class Parc {
    private ArrayList<Vehicule> liste = new ArrayList<>();

    public void addVehicule(Vehicule v) {
        for(Vehicule existant : liste) {
            if(existant.getImmatriculation().equals(v.getImmatriculation())) {
                throw new RuntimeException("Vehicule deja existant");
            }
        }
        liste.add(v);
    }

    public void removeVehicule(String immat) {
        Vehicule vToRemove = null;
        for(Vehicule v : liste) {
            if(v.getImmatriculation().equals(immat)) {
                vToRemove = v;
                break;
            }
        }
        if(vToRemove == null) throw new RuntimeException("Vehicule introuvable");
        liste.remove(vToRemove);
    }

    public double getPrixTotalLocation(int nbJour) {
        double total = 0;
        for(Vehicule v : liste) total += v.getPrixLocation(nbJour);
        return total;
    }
}
Probleme 2

Problème 2 - Tri de nombres

Écrire un programme qui lit une suite de nombres au clavier et qui les affiche ensuite triés par ordre croissant. Le nombre de nombres sera fourni au départ.

Correction et raisonnement

Étape 1 : Utilisation du Scanner et des tableaux.
On importe Scanner pour la lecture et Arrays pour le tri automatique.

import java.util.Scanner; 
import java.util.Arrays;

public class Tri {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        System.out.print("Combien de nombres ? ");
        int nbl = sc.nextInt();
        
        int[] tab = new int[nbl];
        System.out.println("Donnez vos nombres :");
        for(int i=0; i<nbl; i++) {
            tab[i] = sc.nextInt();
        }
        
        Arrays.sort(tab);
        
        System.out.println("Liste par ordre croissant :");
        for(int val : tab) {
            System.out.print(val + " ");
        }
        sc.close();
    }
}
Probleme 3

Problème 3 - Héritage vs Composition

On dispose d'une classe Point(x, y). Écrire une classe Cercle avec constructeur, deplaceCentre, changeRayon, getCentre et affiche.

  • Question 1 : Comme classe dérivée de Point.
  • Question 2 : Possédant un membre de type Point.
Correction et raisonnement

Étape 1 : Héritage (Cercle EST UN Point).

public class Cercle extends Point {
    private double rayon;

    public Cercle(double x, double y, double r) { 
        super(x, y); 
        this.rayon = r; 
    }
    
    public void deplaceCentre(double dx, double dy) { 
        super.deplace(dx, dy); 
    }
    
    public Point getCentre() { 
        return new Point(getX(), getY()); 
    }
}
Correction et raisonnement

Étape 2 : Composition (Cercle A UN Point).

public class Cercle {
    private Point centre;
    private double rayon;

    public Cercle(double x, double y, double r) { 
        this.centre = new Point(x,y); 
        this.rayon = r; 
    }
    
    public void deplaceCentre(double dx, double dy) { 
        centre.deplace(dx, dy); 
    }
    
    public Point getCentre() { 
        return centre; 
    }
}
Probleme 4

Problème 4 - Constructeurs

On donne les classes A et B extends A. Quel est le résultat de l'exécution ?

Correction et raisonnement

Étape 1 : Trace de l'exécution.
Lors de l'appel à new A(5), les attributs sont d'abord initialisés (n=0, p=10) puis le constructeur est exécuté.
Lors de l'appel à new B(5, 3), le constructeur de B appelle super(n) avant d'exécuter le reste de son code. L'attribut q de B est initialisé à 25.

Probleme 5

Problème 5 - Exceptions

On donne un programme effectuant une boucle for(i=0; i<=3; i++) qui appelle une méthode jetant des exceptions Erreur1, Erreur2 et Erreur selon la valeur de i. Quel est le résultat ?

Correction et raisonnement

Étape 1 : Analyse des itérations.
- \(i=0\) : Aucune erreur, imprime "-----".
- \(i=1\) : Erreur1 jetée. Attrapée dans fn(), affiche "fn1". Fin de boucle, affiche "-----".
- \(i=2\) : Erreur2 jetée. Erreur2 hérite d'Erreur. Attrapée dans fn() par le bloc catch(Erreur), affiche "fn0", puis est relancée (throw e;). Attrapée dans le main par le bloc catch(Erreur2), affiche "main2". Affiche "-----".
- \(i=3\) : Erreur jetée. Attrapée dans fn(), affiche "fn0", relancée. Attrapée dans le main par catch(Erreur), affiche "main0". Affiche "-----". A la fin, "fin".

Sortie Console :
-----
fn1
-----
fn0
main2
-----
fn0
main0
-----
fin

Correction et raisonnement

Question 1 :
Compile : Oui. Exécution : Oui. Sortie : B (Polymorphisme : appel de la méthode de l'objet réel).

Question 2 :
Compile : Non. Incompatibilité de type : la référence B (fille) ne peut pas instancier un objet A (mère).

Question 3 :
Compile : Non. La méthode f(int) n'est pas définie dans la classe A (qui est le type de déclaration de la référence).

Question 4 :
Compile : Non. Incompatibilité de type : une référence de type B ne peut pas pointer vers une instance de type A.

Question 5 :
Compile : Oui. Exécution : Non. NullPointerException (Appel de la méthode length() sur une String nulle).

Question 6 :
Compile : Oui. Exécution : Non. Boucle infinie car la variable i n'est jamais incrémentée dans le while.

Question 7 :
Compile : Oui. Exécution : Oui. Sortie : A puis B (Appel à super.afficher() depuis la classe fille).

Question 8 :
Compile : Oui. Exécution : Oui. Sortie : Erreur puis Finally.

Question 9 :
Compile : Non. La variable c est déclarée final et ne peut donc être affectée qu'une seule fois (erreur à la ligne du second c=a+b;).

Question 10 :
Compile : Oui. Exécution : Oui. Sortie :
true / true / true (Un Chien est un Animal, un Mammifere et un Chien).
true / true / true (Idem pour la référence Mammifere).
false (Un Animal pur n'est pas une instance de Chien).

Question 11 :
Compile : Oui. Exécution : Oui. Sortie :

[0, 10, 30, 40]
[1, 11, 31, 41]
