Exercice 1

Exercice 1 (2 points)

Question : Expliquer ce qu'est un objet immutable. Donner 2 exemples de class Java immutable.

Correction et raisonnement

Réponse et Raisonnement : Un objet immutable (immuable en français) est un objet dont l'état (les valeurs de ses attributs) ne peut pas être modifié une fois qu'il a été créé et initialisé.

  • Exemples : String, Integer (ou toute autre classe Wrapper comme Double), et LocalDate (vue dans l'exercice 4).
Exercice 2

Exercice 2 (1 point)

Question : Expliquez pourquoi ce programme ne veut pas se lancer.

Correction et raisonnement

Réponse et Raisonnement : La signature de la méthode principale est incorrecte. Dans le code fourni, la méthode est écrite public void main(String args). Pour que la Machine Virtuelle Java (JVM) puisse identifier le point d'entrée du programme et l'exécuter, la méthode doit obligatoirement être static et prendre en paramètre un tableau de chaînes de caractères. Il faudrait écrire : public static void main(String[] args).

Exercice 3

Exercice 3 (2 points)

Question : Ce programme ne compile pas. a) Expliquez l'erreur b) Proposer un correctif.

Correction et raisonnement

Réponse et Raisonnement :

  • a) Explication : L'erreur provient de la ligne ArrayList<int> maListe = new ArrayList<int>();. En Java, les collections et la généricité ne fonctionnent qu'avec des types objets (références). Il est impossible d'utiliser un type primitif comme int.
  • b) Correctif : Il faut utiliser la classe Wrapper (enveloppe) Integer correspondante au type primitif int.
ArrayList<Integer> maListe = new ArrayList<Integer>();
Exercice 4

Exercice 4 (4 points)

Question a : Expliquer ce que fait la ligne return this (year, month, dayOfMonth);.

Correction et raisonnement

Réponse a : Cette syntaxe, bien qu'incorrecte dans une méthode statique en Java classique (il faudrait un new LocalDate(...)), représente conceptuellement l'appel au constructeur de la classe courante afin d'instancier et de retourner un nouvel objet LocalDate.

Question b : Indiquer la ligne causant une erreur de compilation et en expliquer la raison.

Correction et raisonnement

Réponse b : La ligne en erreur est d = new LocalDate (2023,2,15);. La raison est que le constructeur de la classe LocalDate est déclaré private. Il est donc inaccessible depuis une autre classe (comme le programme principal).

Question c : Si possible proposer un correctif.

Correction et raisonnement

Réponse c : Il faut utiliser les méthodes statiques fournies par la classe pour obtenir une instance (patron de conception Factory / Fabrique).

d = LocalDate.of(2023, 2, 15);

Question d : Comment peut-on qualifier ses 2 méthodes of en utilisant le vocabulaire orienté objet adapté ?.

Correction et raisonnement

Réponse d : C'est de la surcharge (Overloading). On a plusieurs méthodes qui portent le même nom (of) mais qui diffèrent par la signature de leurs paramètres (l'une prend int, int, int et l'autre int, Month, int ).

Exercice 5

Exercice 5 (2 points)

Question 1 : Erreur de compilation sur Bien b = new Bien(...);.

Correction et raisonnement

Réponse 1 : La classe Bien est déclarée comme abstract. Une classe abstraite ne peut pas être instanciée directement avec new.

Question 2 : Concernant la méthode toString, comment qualifier cette méthode ?.

Correction et raisonnement

Réponse 2 : C'est une redéfinition (Overriding). L'annotation @Override indique que la classe Bien fournit sa propre implémentation de la méthode toString() héritée de la super-classe mère universelle Object.

Exercice 6

Exercice 6 (3 points)

Question a : Expliquer la cause de l'erreur sur System.out.println("" + b.getSurface());.

Correction et raisonnement

Réponse a : La variable b est déclarée avec le type de référence Bien. Lors de la compilation, le compilateur vérifie les méthodes disponibles pour le type de référence. Or, la méthode getSurface() n'est pas définie dans la classe Bien , elle n'existe que dans la classe fille Maison.

Question b : Proposer un correctif.

Correction et raisonnement

Réponse b : On peut procéder à un "cast" (transtypage) pour indiquer au compilateur que l'objet pointé est bien une Maison.

System.out.println("" + ((Maison) b).getSurface());
// Alternative : déclarer b en tant que Maison
// Maison b = new Maison(1, "Villa Magnifico", "Valence", 1000);
Exercice 7

Exercice 7 (3 points)

Question 1 : Expliquer l'erreur dans la classe Maison.

Correction et raisonnement

Réponse 1 : La classe Maison hérite de la classe Bien. La classe Bien possède une méthode abstraite addTarif(Tarif newTarif). Puisque Maison est une classe concrète, elle est dans l'obligation d'implémenter (redéfinir) toutes les méthodes abstraites héritées de sa classe mère.

Question 2 : Apporter un correctif (code à ajouter dans la classe Maison).

Correction et raisonnement

Réponse 2 :

@Override
public void addTarif(Tarif newTarif) {
    // Code d'ajout du tarif
}
Exercice 8

Exercice 8 (3 points)

Question 1 : Erreur à l'exécution et correctif.

Correction et raisonnement

Réponse 1 : L'erreur est une NullPointerException. La variable mesBiens est déclarée mais n'a pas été initialisée (elle vaut null). Quand on tente d'appeler mesBiens.add(...), le programme plante.

Correction et raisonnement

Correctif :

ArrayList<Bien> mesBiens = new ArrayList<>();

Question 2 : Compléter le programme pour afficher les maisons dont la ville est "Valence" et leur surface.

Correction et raisonnement

Réponse 2 : Il faut parcourir la liste, vérifier avec instanceof que le bien est une maison, puis vérifier la ville.

for (Bien b : mesBiens) {
    // On verifie que c'est une Maison pour pouvoir caster et acceder a la surface
    if (b instanceof Maison) {
        if ("Valence".equals(b.getVille())) {
            Maison m = (Maison) b;
            System.out.println("Maison : " + m.getNom() + " | Surface : " + m.getSurface());
        }
    }
}
