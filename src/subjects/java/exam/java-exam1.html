Exercice 1

Exercice 1 (3 points)

Question : Quel est le résultat de ce programme?

Correction et raisonnement

Résultat et Raisonnement : Le programme illustre les principes d'héritage et de polymorphisme (liaison dynamique).

  • A a = new A(); a.affiche(); \(\rightarrow\) Je suis un A
  • B b = new B(); b.affiche(); \(\rightarrow\) Je suis un A (B hérite de A et ne redéfinit pas la méthode ).
  • C c = new C(); c.affiche(); \(\rightarrow\) Je suis un C (C redéfinit affiche() ).
  • a = c; a.affiche(); \(\rightarrow\) Je suis un C (Polymorphisme : a pointe vers une instance de C ).
  • D d = new D(); d.affiche(); \(\rightarrow\) Je suis un D (D redéfinit affiche() ).
  • a = d; a.affiche(); \(\rightarrow\) Je suis un D (a pointe vers D ).
  • c = d; c.affiche(); \(\rightarrow\) Je suis un D (c pointe vers D ).
  • E e = new E(); e.affiche(); \(\rightarrow\) Je suis un A (E hérite de B, qui hérite de A. Aucune ne redéfinit affiche() ).
  • a = e; a.affiche(); \(\rightarrow\) Je suis un A (a pointe vers E ).
  • b = e; b.affiche(); \(\rightarrow\) Je suis un A (b pointe vers E ).
  • F f = new F(); f.affiche(); \(\rightarrow\) Je suis un C (F hérite de C qui possède sa propre méthode affiche() ).
  • a = f; a.affiche(); \(\rightarrow\) Je suis un C (a pointe vers F ).
  • c = f; c.affiche(); \(\rightarrow\) Je suis un C (c pointe vers F ).

Question 2.2

Correction et raisonnement

Raisonnement : La méthode equals(Object o) doit être redéfinie pour comparer les valeurs et non les références mémoires.

@Override
public boolean equals(Object obj) {
    if (this == obj) return true;
    if (obj == null || getClass() != obj.getClass()) return false;
    TypPneu typPneu = (TypPneu) obj;
    return diametre == typPneu.diametre && 
           largeur == typPneu.largeur && 
           marque.equals(typPneu.marque);
}

Question 2.6

Correction et raisonnement

Raisonnement : == compare les références mémoires. equals() compare le contenu.

  • p1 == p2 \(\rightarrow\) false (Différentes instances en mémoire).
  • p1 == p3 \(\rightarrow\) false (Différentes instances).
  • p1.equals(p2) \(\rightarrow\) false (Marques différentes : "Michelin" \(\neq\) "Dunlop" ).
  • p1.equals(p3) \(\rightarrow\) false (Dimensions différentes ).
  • p1 == p4 \(\rightarrow\) false (p4 pointe vers p3 ).
  • p3 == p4 \(\rightarrow\) true (Même référence ).
  • p3.equals(p4) \(\rightarrow\) true (Même référence, donc même contenu).
  • v1.getTypPneu() == v2.getTypPneu() \(\rightarrow\) true (Les deux voitures partagent l'objet p1 ).
  • v1.getTypPneu() == v3.getTypPneu() \(\rightarrow\) false (p1 \(\neq\) p3).
  • v1.getTypPneu().equals(p2) \(\rightarrow\) false (Équivaut à p1.equals(p2)).
Exercice 3

Exercice 3 (3 points)

3.1

On crée un objet en utilisant le mot-clé new suivi de l'appel au constructeur de la classe (ex: A a = new A();).

3.2

En Java, le développeur ne supprime pas explicitement les objets. C'est le Garbage Collector (Ramasse-miettes) qui détruit automatiquement les objets en mémoire lorsqu'ils ne sont plus référencés.

3.3

Java est à la fois compilé et interprété. Le code source (.java) est d'abord compilé en un code intermédiaire appelé Bytecode (.class). Ce bytecode est ensuite interprété (et souvent compilé à la volée, JIT) par la Machine Virtuelle Java (JVM) lors de l'exécution.

3.4

Le mot-clé final pour une variable indique qu'elle est une constante. Une fois initialisée, sa valeur ne peut plus être modifiée. Exemple : final double PI = 3.14159; Si on tente de réassigner PI = 3.14;, le compilateur renverra une erreur.

3.5

  • Classe abstraite : Peut contenir des attributs (état), des méthodes concrètes et des constructeurs. Une classe ne peut hériter que d'une seule classe (abstraite ou non).
  • Interface : Ne contient (traditionnellement) que des signatures de méthodes abstraites et des constantes (public static final). Une classe peut implémenter plusieurs interfaces.

Exercice 6 (3 points)

import java.util.ArrayList;
import java.util.List;

public class Parc {
    private List<Vehicule> vehicules;

    public Parc() {
        this.vehicules = new ArrayList<>();
    }

    public void addVehicule(Vehicule v) {
        for (Vehicule existant : vehicules) {
            if (existant.immatriculation.equals(v.immatriculation)) {
                throw new RuntimeException("Vehicule deja existant");
            }
        }
        vehicules.add(v);
    }

    public void removeVehicule(String immat) {
        Vehicule aSupprimer = null;
        for (Vehicule v : vehicules) {
            if (v.immatriculation.equals(immat)) {
                aSupprimer = v;
                break;
            }
        }
        if (aSupprimer == null) {
            throw new RuntimeException("Vehicule introuvable");
        }
        vehicules.remove(aSupprimer);
    }

    public void displayParc() {
        for (Vehicule v : vehicules) {
            System.out.println("Immat: " + v.immatriculation + 
                ", Prix jour: " + v.getPrixLocation(1));
            // L'ideal serait d'avoir une methode toString() dans Vehicule
        }
    }

    public double getPrixTotalLocation(int nbJour) {
        double total = 0;
        for (Vehicule v : vehicules) {
            total += v.getPrixLocation(nbJour);
        }
        return total;
    }
}
